11// MQTT :
12// - Pour "Message Queuing Telemetry Transport", est un protocole open source

15// Adafruit :
16// - Adafruit Industries est une entreprise fondée en 2005 par Limor Fried, une ingénieure américaine,
17// - qui se spécialise dans la vente et la production de composants électroniques et de matériel libre.
18
19// ESP32 :
20// - ESP32 est une famille de microcontrôleurs à base de processeurs 32 bits,
21// - développée par Espressif Systems, une société chinoise basée à Shanghai.
22
23// STATION (WIFI_STA) :
24// - Le mode Station (STA) est utilisé pour connecter le module ESP32 à un point d’accès Wi-Fi.
25// - L’ESP32 se comporte comme un ordinateur qui serait connecté à notre box. Si la box est reliée à Internet, alors l’ESP32 peut accéder à Internet. L’ESP32 peut se comporter en tant que client, c’est-à-dire faire des requêtes aux autres appareils connectés sur le réseau ou en tant que serveur , c’est-à-dire que d’autres appareils connectés sur le réseau vont envoyer des requêtes à l’ESP32. Dans les 2 cas, l’ESP32 peut accéder à Internet.
26
27// AP (Access Point) (WIFI_AP) :
28// - En mode Access Point, l’ESP32 se comporte comme un réseau Wi-Fi (un peu comme une box) :
29// - d’autres appareils peuvent s’y connecter. Dans ce mode, l’ESP32 n’est relié à aucun autre réseau et n’est donc pas connecté à Internet.
30// - Ce mode est plus gourmand en calcul et en énergie (la carte ESP32 va chauffer) puisque l’ESP32 doit simuler un routeur Wifi complet (Soft AP).

49// \section EspConnect 1 : Mise en place d'une connexion entre les esp et le point d'accès :
50 // - Utilisation de la librairie MyWiFi.h (permets la connexion simultanée de plusieurs appareils sur un même access point).
51 // - On utilise en général le mode AP provisoirement pour rentrer les paramètres de connexion du réseau WiFi (SSID + MDP).
52 // - On peut également l’utiliser pour avoir un réseau séparé de son réseau domestique et non relié à Internet.
53 // \subsection fonctions Fonctions Utilisées
54 // - setup Wifi(): Va permettre de setup l’Access point et vérifier si l'utilisateur en mode station peut se connecter.
55 // - Connect_WiFi_AP(const char* ssid, const char* password): va permettre de connecter l'esp au point d'accès.
56 // - Check_WiFi_connection(): permet de vérifier si l'esp est connecté au point d'accès.
